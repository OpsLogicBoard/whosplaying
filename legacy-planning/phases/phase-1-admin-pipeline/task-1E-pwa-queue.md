## Objective

## Sub-tasks

### 1E.1 — PWA shell
- File: `apps/admin/next.config.js` — add `next-pwa` config
- Files: `apps/admin/public/manifest.json` + icon assets (192, 512, maskable)
- Service worker generated by `next-pwa`
- Verify: open `admin.whosplaying.live` on iOS Safari → "Add to Home Screen" works → app icon appears, opens fullscreen

### 1E.2 — Web push setup
- Generate VAPID keys: `npx web-push generate-vapid-keys`
- Store as env vars: `VAPID_PUBLIC_KEY` (also `NEXT_PUBLIC_`), `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT="mailto:whosplayingjaxbeach@gmail.com"`
- Endpoint: `apps/admin/app/api/push/subscribe/route.ts` — accepts a `PushSubscription` JSON, stores in `settings.push_subscription_json`
- Endpoint: `apps/admin/app/api/push/send/route.ts` — server-side send via `web-push`, called by `whosplaying:generate-post` Skill

### 1E.3 — `/queue` page (the swipe deck)
- File: `apps/admin/app/queue/page.tsx`
- Mobile-first: fullscreen card stack, no chrome
- Card front: screenshot (or placeholder), parsed Artist @ Venue, time, source platform pill
- Gesture lib: `framer-motion` `<motion.div drag />`
- Right swipe (≥30% threshold) → POST `/api/captures/[id]/approve` → calls `match-capture` re-eval (which now writes events + calendar) — actually since match-capture is webhook-driven on insert, instead this just updates capture_status to 'approved' and triggers events insert directly via a server action
- Left swipe → POST `/api/captures/[id]/reject`
- Tap = open edit panel (parsed fields editable, "Save & Approve")
- "Auto-published today" section pinned at bottom: scrollable list of events from `auto_publish_log` for `digest_date=today`, each with one-tap **Undo** button (calls `calendar-push` with action=delete + soft-delete event + flips `social_captures.capture_status='rejected'`)
- Empty state: "Queue clear. 🎉 12 events auto-published today."

### 1E.4 — Server actions for queue mutations
- `apps/admin/app/queue/actions.ts`:
  - `approveCapture(id)` — creates events + event_artists, links capture, fires calendar-push
  - `rejectCapture(id, reason?)` — sets status, no calendar action
  - `editAndApprove(id, patch)` — same as approve but with overrides
  - `undoAutoPublish(eventId)` — soft-deletes event, calls calendar-push delete, marks `auto_publish_log.undone_at`
  - `addCanonicalVenue({ name, aliases })` — appends row to Venues Sheet via Sheets API, triggers immediate `sheets-sync`

### 1E.5 — `/settings` page
- File: `apps/admin/app/settings/page.tsx`
- Toggles: `auto_approve_enabled`, `scheduled_sweep_enabled`
- Time inputs: sweep times, post generation time, digest push time
- Slider: `min_vision_confidence`
- Buttons: "Sync Sheets now", "Run sweep now", "Test push notification"
- Subscribe button: registers service worker push subscription, POSTs to `/api/push/subscribe`
